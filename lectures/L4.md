# Лекция 4. Эквивалентные коды. Циклические коды. Полиномиальное представление. Решётчатые представления (trellis)

### 1. Эквивалентные коды

В теории кодирования разные коды могут задаваться по-разному (например, различными порождающими матрицами), но при этом обладать одинаковыми характеристиками. Такие коды называют **эквивалентными**.

**Определение.**  
Два линейных кода считаются **эквивалентными**, если один получается из другого с помощью **перестановки позиций символов** в кодовых словах (то есть перестановкой столбцов матрицы $G$).

**Свойства эквивалентных кодов:**

- Совпадают параметры: длина $n$, размерность $k$.  
- Одинаковое минимальное расстояние $d_{\min}$.  
- Спектр весов (распределение кодовых слов по весам) совпадает.  

**Важное замечание.**  
Достаточно исследовать один код из класса эквивалентности, чтобы получить информацию обо всех остальных. Поэтому часто матрицу $G$ приводят к **систематическому виду** $[I_k | P]$ — такой вид эквивалентен исходному и удобен в работе.

**Пример.**  
Перестановка столбцов в матрице $G$ не меняет корректирующие свойства кода, но изменяет порядок символов в кодовых словах.

### 2. Циклические коды: Определение и структура

Циклические коды представляют собой важный класс линейных блоковых кодов, обладающий дополнительной структурой, упрощающей операции кодирования и декодирования.

**Определение.**  
Линейный $(n, k)$-код называется **циклическим**, если любой циклический сдвиг кодового слова $c = (c_0, c_1, \dots, c_{n-1}) \in C$ также принадлежит $C$.  
Например, сдвиг вправо: $c^{(1)} = (c_{n-1}, c_0, c_1, \dots, c_{n-2}) \in C$.

**Алгебраическая структура.**  
Такие коды образуют **идеал** в кольце многочленов по модулю $x^n - 1$ (над GF(2)).

### 3. Полиномиальное представление кодовых слов

Для циклических кодов удобно переходить от векторной записи к представлению через **многочлены** над GF(2):

**Соответствие:**  
$\vec{c} = (c_0, c_1, \dots, c_{n-1}) \leftrightarrow c(x) = c_0 + c_1 x + c_2 x^2 + \dots + c_{n-1} x^{n-1}$.

**Свойство циклического сдвига:**  
Сдвиг вправо соответствует умножению на $x$ по модулю $x^n - 1$:  
$$x \cdot c(x) \pmod{x^n - 1} = c_{n-1} + c_0 x + c_1 x^2 + \dots + c_{n-2} x^{n-1}.$$

Это гарантирует сохранение принадлежности коду при сдвиге.

### 4. Порождающий многочлен $g(x)$

В каждом циклическом коде существует **единственный** ненулевой многочлен минимальной степени $g(x)$ — **порождающий многочлен**.

**Свойства порождающего многочлена $g(x)$:**

1. $g(x)$ — **делитель** многочлена $x^n - 1$.  
2. Степень $\deg(g(x)) = n - k$ (число проверочных символов).  
3. Любое кодовое слово $c(x)$ представимо как  
   $$c(x) = m(x) \cdot g(x),$$  
   где $m(x)$ — информационный многочлен степени $< k$.

**Теорема.**  
Код является циклическим тогда и только тогда, когда он задаётся одним многочленом $g(x)$.

### 5. Несистематическое кодирование

**Процесс:**  
- Информационное слово $m = (m_0, \dots, m_{k-1}) \leftrightarrow m(x)$.  
- Выполнить умножение: $c(x) = m(x) \cdot g(x)$.  
- Коэффициенты полученного многочлена образуют кодовое слово.

**Пример.**  
Если $g(x) = 1 + x + x^3$, $m(x) = 1 + x$, то  
$c(x) = (1 + x) \cdot (1 + x + x^3) = 1 + x + x + x^2 + x^3 + x^4 = 1 + 0x + x^2 + x^3 + x^4$ (по mod 2).

### 6. Систематическое кодирование

Чтобы информационные символы явно присутствовали в кодовом слове (например, в начале), применяется деление с остатком.

**Алгоритм систематического кодирования:**

1. Умножить $m(x)$ на $x^{n-k}$: $x^{n-k} \cdot m(x)$.  
2. Выполнить деление:  
   $$x^{n-k} \cdot m(x) = q(x) \cdot g(x) + r(x),$$  
   где $\deg(r(x)) < n-k$.  
3. Получить кодовое слово:  
   $$c(x) = x^{n-k} \cdot m(x) + r(x).$$  

(Так как $c(x) \equiv 0 \pmod{g(x)}$, оно принадлежит коду.)

**Преимущество:** информационные биты расположены в начале, проверочные — в конце.

### 7. Проверочный многочлен $h(x)$

Определяется **проверочный многочлен** $h(x)$:

$$g(x) \cdot h(x) = x^n - 1,$$

где $\deg(h(x)) = k$.

**Свойство проверки:**  
$c(x)$ является кодовым словом тогда и только тогда, когда  

$$c(x) \cdot h(x) \equiv 0 \pmod{x^n - 1}.$$

Это аналог проверки синдрома в матричной записи.

### 8. Аппаратная реализация: Линейные регистры сдвига с обратной связью (LFSR)

Циклические коды удобно реализуются аппаратно с помощью **LFSR** (Linear Feedback Shift Registers).

**Кодирование с использованием LFSR:**  
- Регистр выполняет деление на $g(x)$.  
- В него последовательно подаются информационные биты.  
- После обработки $k$ битов в регистре остаётся остаток $r(x)$ (проверочные биты).  

**Декодирование (синдром):**  
- Принятое слово $y(x)$ подаётся в аналогичный регистр.  
- Остаток на выходе — синдром ошибки.

**Преимущество:** простая реализация на сдвиговых регистрах и XOR-элементах.

### 9. Решётчатое представление (Trellis) блоковых кодов

Любой линейный блоковый код можно задать в виде **решётки (trellis)** — ориентированного графа состояний, используемого, например, в алгоритме Витерби.

**Структура решётки:**

- **Уровни:** $n+1$ уровней (от 0 до $n$).  
- **Вершины:** на уровне $i$ — состояния, соответствующие возможным префиксам длины $i$.  
- **Рёбра:** переходы между состояниями, помеченные символами (0 или 1).  
- **Путь:** от начальной вершины (уровень 0) до конечной (уровень $n$) задаёт кодовое слово.  

**Построение решётки (алгоритм):**

1. **Уровень 0:** одна вершина (пустое прошлое).  
   - Будущее: все кодовые слова.  

2. **Для уровня $i$ (от 1 до $n$):**  
   - Определить все возможные префиксы длины $i$.  
   - Для каждого префикса найти соответствующие суффиксы.  
   - Задать вершины (по числу уникальных префиксов).  
   - Соединить вершины рёбрами с предыдущим уровнем, помечая их символами.  

3. **Схлопывание путей (survivor paths):**  
   Если несколько путей приводят к одному состоянию, сохраняется путь с минимальной метрикой.  

**Пример построения (код из лекции):**  

Кодовые слова:  
- c₀ = 000000  
- c₁ = 101010  
- c₂ = 110100  
- c₃ = 011110  
- c₄ = 100110  
- c₅ = 001100  
- c₆ = 010000  
- c₇ = 111010  

**Уровень 0:** Вершина 1. Прошлое: пусто. Будущее: все слова.  

**Уровень 1:** Префиксы: 0 и 1.  
- 0: будущие — c₀, c₃, c₅, c₆.  
- 1: будущие — c₁, c₂, c₄, c₇.  
- Вершины: 1 (0), 2 (1).  
- Переходы: 0 → вершина 1, 1 → вершина 2.

**Уровень 2:** Префиксы: 00, 01, 10, 11.  
- 00: c₀, c₆.  
- 01: c₃, c₅.  
- 10: c₁, c₇.  
- 11: c₂, c₄.  
- Вершины: 1 (00), 2 (01), 3 (10), 4 (11).  
- Переходы:  
  - Из 1 → (00, 0), (01, 1).  
  - Из 2 → (10, 0), (11, 1).

**Визуализация решётки (текстовый вид):**
```
Уровень 0: Вершина 1
    |-0-> Вершина 1 (уровень 1, прошлое: 0)
    |-1-> Вершина 2 (уровень 1, прошлое: 1)

Уровень 1: Вершина 1 (0)
    |-0-> Вершина 1 (уровень 2, прошлое: 00)
    |-1-> Вершина 2 (уровень 2, прошлое: 01)
    Вершина 2 (1)
    |-0-> Вершина 3 (уровень 2, прошлое: 10)
    |-1-> Вершина 4 (уровень 2, прошлое: 11)
```


(На следующих уровнях число состояний может уменьшаться из-за совпадения префиксов.)

**Схлопывание путей:**  
Если несколько путей приходят в одно состояние с одинаковым прошлым — выбирается путь с наименьшей метрикой.
