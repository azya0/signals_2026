# Лекция 3. Синдромное декодирование линейных кодов. Смежные классы. Границы параметров кодов (Хэмминга и Варшамова–Гилберта)

## 1. Постановка задачи декодирования

При передаче информации по двоичному симметричному каналу (ДСК) кодовое слово $c \in C$ может исказиться из-за вектора ошибки $e$. В результате на приёмной стороне наблюдается вектор:

$$y = c \oplus e$$

**Задача декодирования по максимуму правдоподобия (MLD):** определить такое кодовое слово $\hat{c}$, при котором расстояние Хэмминга $d(y, \hat{c})$ минимально. В двоичном случае это равносильно поиску вектора ошибки $\hat{e}$ с наименьшим весом Хемминга $\omega(e)$, для которого выполняется $y = \hat{c} \oplus \hat{e}$.

## 2. Смежные классы (Косеты) и Стандартная таблица

Для упрощения поиска ошибки всё пространство $F_2^n$ (размера $2^n$) разбивается на непересекающиеся подмножества относительно кода $C$.

**Определение.** Смежный класс (косет) кода $C$ по вектору $a \in F_2^n$ — это множество:

$$a \oplus C = \{ a \oplus c \mid c \in C \}$$

### Свойства смежных классов:

1. **Разбиение:** Пространство $F_2^n$ представляется в виде $2^{n-k}$ непересекающихся смежных классов.
2. **Мощность:** В каждом классе содержится ровно $|C| = 2^k$ элементов.
3. **Лидер класса:** Вектор с минимальным весом Хемминга в данном классе. Он соответствует наиболее вероятной ошибке для этого класса.

### Стандартная таблица декодирования (Standard Array)

Этот метод позволяет наглядно представить все возможные векторы пространства, сгруппированные по ошибкам.

| Лидеры ($e$) | $c_0 = 0...0$ | $c_1$ | ... | $c_{2^k-1}$ |
| --- | --- | --- | --- | --- |
| **$e_0 = 0$** | $c_0$ (весь код $C$) | $c_1$ | ... | $c_{2^k-1}$ |
| **$e_1$** | $e_1 \oplus c_0$ | $e_1 \oplus c_1$ | ... | $e_1 \oplus c_{2^k-1}$ |
| ... | ... | ... | ... | ... |
| **$e_{2^{n-k}-1}$** | $e_{last} \oplus c_0$ | ... | ... | $e_{last} \oplus c_{2^k-1}$ |

## 3. Синдромное декодирование

Хранение стандартной таблицы требует $2^n$ ячеек памяти, что на практике неудобно. Синдромное декодирование позволяет уменьшить объём до $2^{n-k}$ записей.

**Определение.** Синдром принятого вектора $y$ — это вектор $S$ длины $r = n - k$:

$$S = y \cdot H^T$$

### Фундаментальные свойства синдрома:

1. **Зависимость только от ошибки:** $S = (c \oplus e)H^T = cH^T \oplus eH^T = \mathbf{0} \oplus eH^T$. Следовательно, синдром не зависит от переданного кодового слова.
2. **Инвариантность в классе:** Все элементы одного смежного класса имеют одинаковый синдром.
3. **Биекция:** Синдромы однозначно соответствуют лидерам смежных классов.


### **Практический алгоритм (Offline + Online)**

**Offline (выполняется один раз):**
- Для всех 2ⁿ возможных векторов **e** вычислить **S = e · Hᵀ**.
- Разбить векторы по значениям синдрома.
- Для каждого синдрома выбрать **лидер** — вектор с минимальным весом.

**Online (при получении y):**
- Вычислить **S = y · Hᵀ**.
- Найти по таблице соответствующий лидер **e**.
- Исправить: **ĉ = y ⊕ e**.

**Преимущество:** требуется хранить только $2^{n−k}$ записей вместо $2^{n}$.

## 4. Геометрическая интерпретация и сферы Хэмминга

Если рассматривать кодовые слова как точки в n-мерном пространстве, то процесс декодирования можно интерпретировать как определение, к какому центру ближе всего принятый вектор.

* **Сфера Хэмминга** радиуса $t$ вокруг слова $c$: множество всех векторов $y$, для которых $d(c, y) \le t$.
* **Объем сферы $V(n, t)$:** число векторов внутри сферы радиуса $t$:

$$V(n, t) = \sum_{i=0}^{t} \binom{n}{i}$$

**Радиус покрытия** — максимальный вес ошибки, который ещё корректируется данным кодом.

**Идеальный случай** — **плотная упаковка сфер**: сферы не пересекаются и полностью покрывают пространство.

## 5. Границы параметров кодов

### 5.1. Граница Хэмминга (Верхняя граница / Плотная упаковка)

Количество кодовых слов $M$ ограничено суммарным объёмом непересекающихся сфер:

$$M \cdot \sum_{i=0}^{t} \binom{n}{i} \le 2^n$$

**Совершенные коды** — это коды, для которых выполняется **равенство** (сферы полностью покрывают пространство).

**Примеры совершенных кодов:**
- Коды Хэмминга (t = 1).
- Код Голея (23,12,7), исправляющий t = 3 ошибки.

**Проверка кода Голея:**

$$2^{12} \cdot (\binom{23}{0} + \binom{23}{1} + \binom{23}{2} + \binom{23}{3}) = 2^{12} \cdot 2048 = 2^{23}$$

Равенство выполняется $\implies$ код Голея является совершенным.

### 5.2. Граница Варшамова–Гилберта (Нижняя граница / Конструктивная)

Данная граница показывает, что коды с заданными параметрами существуют. Линейный $(n, k)$-код с минимальным расстоянием $d$ существует, если:

$$2^{n-k} > \sum_{i=0}^{d-2} \binom{n-1}{i}$$

**Алгоритм построения:**

Проверочная матрица $H$ формируется по столбцам. Новый $i$-й столбец добавляется только в том случае, если он **не выражается** как линейная комбинация любых $(d-2)$ уже выбранных столбцов. Если число «запрещённых» комбинаций меньше общего числа ненулевых векторов $2^{n-k}-1$, то подходящий столбец всегда можно подобрать.

---

## 6. Резюме: Сравнение границ

* **Граница Хэмминга:** показывает, что кодов с лучшими параметрами существовать **не может** (необходимое условие).
* **Граница Варшамова–Гилберта:** гарантирует существование кода с заданными параметрами (достаточное условие).
