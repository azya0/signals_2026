# Лекция 1. Основы канального кодирования

## Общая идея курса
Данный курс посвящён методам передачи данных по каналам с помехами. Основная задача канального кодирования — введение избыточности, позволяющей обнаруживать и исправлять ошибки, возникающие в процессе передачи.

---

## Модель цифровой системы связи

В простейшем виде система цифровой связи может быть представлена как последовательность функциональных блоков:

```
Источник данных 
      |
Канальный кодер 
      |
Канал передачи с шумом 
      |
Декодер 
      |
Получатель информации
```

### Назначение элементов системы
- **Источник данных** формирует информационные символы (как правило, двоичные).
- **Кодер** преобразует информационную последовательность в кодовую, добавляя избыточность.
- **Канал** искажает передаваемые символы вследствие шумов.
- **Декодер** по принятой последовательности восстанавливает исходную информацию.

---

## Канал передачи как математическая модель

В рамках курса рассматриваются дискретные каналы над полем GF(2). Наиболее важным примером является двоичный симметричный канал.

### Двоичный симметричный канал (ДСК)

Для каждого передаваемого бита возможны следующие события:

```
0 → 0  с вероятностью (1 − p)
0 → 1  с вероятностью p
1 → 1  с вероятностью (1 − p)
1 → 0  с вероятностью p
```

Здесь **p** — вероятность ошибки передачи одного символа.  
Например, при p = 10^(-3) в среднем искажается один бит из тысячи.

## Канальные коды: примеры

### Код тройного повторения
Данный код отображает один информационный бит в три кодовых:

| Информация | Код |
|-----------|-----|
| 0         | 000 |
| 1         | 111 |

Декодирование производится по принципу большинства. Код способен исправлять одну ошибку и имеет скорость R = 1/3.

### Пример линейного (5,2)-кода

| Информационное слово | Кодовое слово |
|---------------------|---------------|
| 00                  | 00000         |
| 01                  | 01101         |
| 10                  | 10110         |
| 11                  | 11011         |

Этот код также корректирует одну ошибку, но обладает большей скоростью передачи: R = 2/5.

## Параметры и свойства кодов

Для любого блочного кода вводятся следующие характеристики:
- **N** — длина кодового слова;
- **K** — число информационных символов;
- **R = K/N** — скорость кода;
- **dₘᵢₙ** — минимальное расстояние Хэмминга.

Минимальное расстояние определяет способность кода к обнаружению и исправлению ошибок.

---

## Метрики Хэмминга

- **Вес Хэмминга** — количество единиц в двоичном векторе.
- **Расстояние Хэмминга** — число позиций, в которых два вектора различаются.

Для двоичных векторов расстояние определяется как вес их поразрядной суммы по mod 2.

Корректирующая способность кода удовлетворяет неравенству:
t <= ⌊(dₘᵢₙ − 1)/2⌋.

---

## Линейные коды

Код называется линейным, если сумма любых двух его кодовых слов также является кодовым словом.

Для линейных кодов минимальное расстояние равно минимальному ненулевому весу кодовых слов.

### Матричное описание
- **Порождающая матрица G (K × N)** задаёт правило кодирования: c = m * G.
- **Проверочная матрица H ((N−K) × N)** используется для проверки принадлежности слова коду: c * Hᵀ = 0.

---

## Теорема Шеннона

Для двоичного симметричного канала пропускная способность определяется формулой:

C = 1 − h₂(p),  
где h₂(p) = −p log₂ p − (1 − p) log₂ (1 − p)

Согласно теореме Шеннона:
- при R < C возможно обеспечить сколь угодно малую вероятность ошибки;
- при R > C надёжная передача невозможна.

Например, при p = 10^(-3) пропускная способность составляет примерно 0.988, что означает необходимость очень небольшой избыточности.
